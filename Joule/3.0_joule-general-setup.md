---

# 2. Admin / Tenant Prerequisites

Usually handled by platform/admin team.

## Required

- Joule subscription instance created
- SAP product supporting Joule integration available
- Identity Authentication integrated
- SAP BTP global account available
- Same Cloud Identity tenant configured
- Required BTP entitlements available

---

# Important Services

- SAP BTP
- Cloud Identity Services
- Identity Authentication
- SAP Build Work Zone
- Cloud Foundry

---

# Required Roles

| Role | Purpose |
|---|---|
| `capability_developer` | Build/compile artifacts |
| `capability_release_admin` | Deploy/configure artifacts |
| `end_user` | Access Joule Web Client |

---

# 3. Development Approaches

## Option 1 — Joule Studio CLI

### Best For
- terminal-based development
- Git workflows
- CI/CD pipelines
- automation

### Common Workflow

```bash
joule login
joule status
joule compile
joule lint
joule deploy
```

---

## Option 2 — Joule Studio Code Editor

VS Code extension/tooling for Joule.

### Features
- templates
- validation
- quick fixes
- linting
- deployment
- assistant management

### Best For
- beginners
- guided development
- visual workflows

---

# 4. Important Concepts

| Concept | Meaning |
|---|---|
| Capability | Functionality exposed by Joule |
| DTA | Design Time Artifact |
| RTA | Runtime Artifact |
| `.daar` | Generated runtime artifact |
| Tenant | Customer/environment isolation |

---

# 5. Key Understanding

- Joule development is capability-driven.
- CLI = enterprise/devops focused.
- Code Editor = easier onboarding.
- Identity setup is critical.
- BTP + Cloud Foundry basics are important.
