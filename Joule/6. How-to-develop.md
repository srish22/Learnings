## Core Concepts

| Concept | Meaning |
| :--- | :--- |
| Assistant | Collection of capabilities assembled together |
| Capability | Group of related business skills + language understanding |
| Scenario | Semantic representation of a specific use case |
| Dialog Function | Executable block that performs tasks and returns data/UI |
| DTA (Design Time Artifact) | YAML-based source definition of capabilities/resources |

<br>

---

### DTA

| What Is A DTA? | What Does It Contain? | Why Is It Important? | Convention Over Configuration |
| :--- | :--- | :--- | :--- |
| - Full form в†’ Design Time Artifact<br><br>- Set of structural requirements developers follow while creating artifacts<br><br>- YAML-based source artifact<br><br>- Human-readable capability definition | - Dataset<br><br>- Dialog flow<br><br>- Configurations<br><br>- Assistant configuration YAML | - Treated like source code<br><br>- Git/version control friendly<br><br>- Supports collaboration<br><br>- Enables merge/conflict handling<br><br>- Provides audit trail | - Joule expects predefined folder structure and naming conventions<br><br>- Developers follow standard structure instead of manually configuring everything |

 
 <br>
 
**Types Of Design Time Artifacts (DTA)**

| DTA | Description |
| :--- | :--- |
| Capability | Contains dataset, conversation flow, dialog functions and capability configuration. |



**Note:** New features are implemented with new versions of the DTA Specification. To use new features, you must update the version in the capability.sapdas.yaml file to the corresponding version. 

<br>

---

### YAML

| What Is YAML? | Why Developers Use It | Common Uses | Important Rules |
| :--- | :--- | :--- | :--- |
| - Human-readable data format<br><br>- Used for configuration and structured data<br><br>- Uses indentation instead of brackets | - Easy to read/write<br><br>- Lightweight syntax<br><br>- Git-friendly<br><br>- Easy to maintain | - Joule DTAs<br><br>- CI/CD pipelines<br><br>- Kubernetes configs<br><br>- GitHub Actions<br><br>- Application settings | - Use spaces, not tabs<br><br>- Indentation matters<br><br>- Structure is hierarchy-based<br><br>- Keep formatting clean |

## Example Assistant Configuration -  *.da.sapdas.yaml

```yaml
schema_version: 1.0.0 # DA Config Schema Version

name: MyTestDigitalAssistant # Name of Digital Assistant

capabilities:
  - type: local
    folder: ./my_capability # Local capability DTA

  - type: local
    folder: ./my_second_capability

  - type: local
    path: ./my_third_capability/my_third_capability.daar
    # Local precompiled .daar file

conversational_search:
  enabled: true

  product_filter:
    - SAP_SUCCESSFACTORS
```

## Description of each

| File / Folder | Mandatory | Purpose |
| :--- | :--- | :--- |
| `capability.sapdas.yaml` | **Mandatory** | Main capability configuration file to define the metadata and configurations|
| `deployment_extension.yaml` | Optional | Used for system context filtering and deployment control |
| `capability_context.yaml` | Optional | Defines statically referenced capability context variables |
| `functions/` | Required if scenario type is function to execute the functionality | Stores dialog functions |
| `hooks/` | Optional | Stores lifecycle functions like `initialize.yaml` |
| `scenarios/` | **Mandatory** | Define the functionality , can be of type function |
| `i18n/` | Optional | Stores localization/property files |
| `agents/` | Optional | Stores agent definitions and schema files |

> Resource Limits for Capability Files: [Official Documentation](https://help.sap.com/docs/joule/joule-development-guide-ba88d1ec6a1b442098863d577c19b0c0/resource-limits-for-capability-files)

## Steps to create a Capability

| Step | Action |
| :--- | :--- |
| 1 | Create capability root folder |
| 2 | Create required folders like `scenarios/`, `functions/`, `agents/`, `i18n/` |
| 3 | Create `capability.sapdas.yaml` configuration file |
| 4 | Add scenarios and dialog functions |
| 5 | Configure in assistant inside `*.da.sapdas.yaml` |
